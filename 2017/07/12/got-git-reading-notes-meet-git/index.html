读书笔记，记录《Git 权威指南》第一章里有价值的命令和概念。

diff 与 patch#

Git 出现之前，diff 和 patch 就已是源码协作的基石，至今仍在使用。生成差异：

diff -u hello.old hello.new | less -N
  • -u：unified 格式，输出带上下文，是生成补丁的必备参数。
  • less -N：分页并显示行号，便于引用具体行讨论。

把差异保存为补丁文件再分发给他人：

diff -u hello.old hello.new > hello.patch

hello.patch 内容形如：

--- hello.old   2026-05-14 10:00:00
+++ hello.new   2026-05-14 10:05:00
@@ -1,3 +1,3 @@
 hello
-world
+git
 !

对方拿到 hello.old 和 hello.patch，用 patch 反向打回去：

patch hello.old < hello.patch       # 把 hello.old 升级成 hello.new
patch -R hello.old < hello.patch    # -R 撤销，回到 hello.old 原状
patch -p1 < changes.patch           # -p1 跳过路径首层，用于跨目录补丁

Git 的 git diff / git apply / git format-patch / git am 本质上就是这套机制的强化版。

分布式 vs 集中式#

集中式（CVS / SVN）所有写操作依赖中心仓库；Git 的克隆即完整版本库，提交、查日志、建分支、合并、回退全部本地完成，无需联网。这也是 Git “快"的根因。

实用命令#

重写最近一次提交#

git commit --amend          # 不带 -m 会进入编辑器修改提交说明

撤销误提交的大文件#

提交里混进了 1 GB 的虚拟机镜像：

git rm --cached winxp.img   # 从索引移除，保留工作区文件
git commit --amend

暂存进度去救火#

git stash
git checkout <hotfix-branch>
# fix something
git checkout -
git stash pop

提交粒度#

一次提交只做一件事。下班前把一天改动一锅端，等于把 Git 降级为文件备份系统。

安装#

# macOS
brew install git

# Debian/Ubuntu
sudo apt install git git-doc git-svn git-email gitk

git-svn git-email gitk 因依赖不同（perl、tk）单独打包；git-doc 安装后可用浏览器查帮助：

git help -w <sub-command>

中文路径正常显示#

避免 git status 把中文文件名转义成八进制：

git config --global core.quotepath false

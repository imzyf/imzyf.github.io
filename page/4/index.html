经典 Git 协同模型

集中式协同模型

可以像集中式版本控制系统那样使用 Git，在一个大家都可以访问到的服务器上架设 Git 服务器，每个人从该服务器克隆代码，本地提交推送到服务器上。

金字塔式协同模型

虽然理论上每个开发者的版本库都是平等的，但是会有一个公认的权威的版本库，这个版本库由一个或者多个核心开发者负责维护（具有推送的权限）。

开源社区逐渐发展出金字塔模型，而这也是必然之选。
